## 3. Table Definitions

### 3.1 `saved_queries` Table

```sql
-- Create the saved_queries table
CREATE TABLE public.saved_queries (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    query_text TEXT NOT NULL,
    explanation TEXT NOT NULL,
    complexity INTEGER NOT NULL CHECK (complexity >= 1 AND complexity <= 10),
    optimization_tips TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Add comments for documentation
COMMENT ON TABLE public.saved_queries IS 'Stores SQL queries analyzed and saved by authenticated users';
COMMENT ON COLUMN public.saved_queries.id IS 'Unique identifier for each saved query';
COMMENT ON COLUMN public.saved_queries.user_id IS 'References the authenticated user who saved this query';
COMMENT ON COLUMN public.saved_queries.query_text IS 'The original SQL query text (max 5000 characters)';
COMMENT ON COLUMN public.saved_queries.explanation IS 'AI/rule-generated plain-English explanation';
COMMENT ON COLUMN public.saved_queries.complexity IS 'Complexity score from 1 (simple) to 10 (expert)';
COMMENT ON COLUMN public.saved_queries.optimization_tips IS 'JSON string array of optimization suggestions (nullable)';
COMMENT ON COLUMN public.saved_queries.created_at IS 'Timestamp of when the query was saved';
```

### 3.2 Column Specifications

| Column | Type | Nullable | Default | Constraints | Description |
|---|---|---|---|---|---|
| `id` | `UUID` | No | `gen_random_uuid()` | PRIMARY KEY | Auto-generated unique identifier |
| `user_id` | `UUID` | No | — | FK → `auth.users(id)` ON DELETE CASCADE | Owner of the saved query |
| `query_text` | `TEXT` | No | — | Length ≤ 5000 (application-enforced) | Original SQL query string |
| `explanation` | `TEXT` | No | — | — | Plain-English explanation generated by the parser |
| `complexity` | `INTEGER` | No | — | CHECK (1–10) | Complexity score |
| `optimization_tips` | `TEXT` | Yes | `NULL` | — | JSON-serialized array of optimization tips |
| `created_at` | `TIMESTAMPTZ` | No | `NOW()` | — | UTC timestamp of creation |

## 4. Indexes

```sql
-- Primary key index (automatically created)
-- idx_saved_queries_pkey ON saved_queries(id)

-- Index for user-specific queries (most common access pattern)
CREATE INDEX idx_saved_queries_user_id ON public.saved_queries(user_id);

-- Index for user + created_at (for ordered history retrieval)
CREATE INDEX idx_saved_queries_user_created ON public.saved_queries(user_id, created_at DESC);
```

### 4.1 Index Justification

| Index | Query Pattern | Benefit |
|---|---|---|
| `idx_saved_queries_user_id` | `WHERE user_id = ?` | Fast lookup of all queries for a user |
| `idx_saved_queries_user_created` | `WHERE user_id = ? ORDER BY created_at DESC` | Fast sorted retrieval for history page |

## 5. Row-Level Security (RLS)

### 5.1 Enable RLS

```sql
-- Enable RLS on the saved_queries table
ALTER TABLE public.saved_queries ENABLE ROW LEVEL SECURITY;
```

### 5.2 RLS Policies

```sql
-- Policy: Users can only SELECT their own saved queries
CREATE POLICY "Users can view own saved queries"
    ON public.saved_queries
    FOR SELECT
    USING (auth.uid() = user_id);

-- Policy: Users can only INSERT queries for themselves
CREATE POLICY "Users can insert own saved queries"
    ON public.saved_queries
    FOR INSERT
    WITH CHECK (auth.uid() = user_id);

-- Policy: Users can only DELETE their own saved queries
CREATE POLICY "Users can delete own saved queries"
    ON public.saved_queries
    FOR DELETE
    USING (auth.uid() = user_id);

-- Note: No UPDATE policy — saved queries are immutable once created
```

### 5.3 RLS Policy Matrix

| Operation | Policy Name | Condition | Effect |
|---|---|---|---|
| `SELECT` | "Users can view own saved queries" | `auth.uid() = user_id` | Users see only their own rows |
| `INSERT` | "Users can insert own saved queries" | `auth.uid() = user_id` | Users can only insert rows with their own user_id |
| `DELETE` | "Users can delete own saved queries" | `auth.uid() = user_id` | Users can only delete their own rows |
| `UPDATE` | (No policy) | — | Updates are not permitted (queries are immutable) |

### 5.4 RLS Testing Queries

```sql
-- Test 1: Verify RLS is enabled
SELECT tablename, rowsecurity 
FROM pg_tables 
WHERE tablename = 'saved_queries';
-- Expected: rowsecurity = true

-- Test 2: Verify policies exist
SELECT policyname, cmd, qual 
FROM pg_policies 
WHERE tablename = 'saved_queries';
-- Expected: 3 policies (SELECT, INSERT, DELETE)

-- Test 3: Anonymous access should return no rows
-- (Run without auth context)
SELECT * FROM public.saved_queries;
-- Expected: 0 rows

-- Test 4: Authenticated access should return only user's rows
-- (Run with auth context of user A)
SELECT * FROM public.saved_queries;
-- Expected: Only user A's rows
```

## 6. Data Operations

### 6.1 INSERT — Save a Query

```typescript
// Client-side (via Supabase SDK)
const { data, error } = await supabase
  .from('saved_queries')
  .insert({
    user_id: session.user.id,
    query_text: queryText,
    explanation: explanation,
    complexity: complexityScore,
    optimization_tips: JSON.stringify(tips),
  })
  .select()
  .single();
```

### 6.2 SELECT — Fetch User's History

```typescript
// Fetch all saved queries for the authenticated user (ordered by newest first)
const { data, error } = await supabase
  .from('saved_queries')
  .select('*')
  .order('created_at', { ascending: false });
// RLS automatically filters to current user's rows
```

### 6.3 SELECT — Fetch Recent Queries (Dashboard)

```typescript
// Fetch the 3 most recent saved queries
const { data, error } = await supabase
  .from('saved_queries')
  .select('*')
  .order('created_at', { ascending: false })
  .limit(3);
```

### 6.4 SELECT — Fetch Stats (Dashboard)

```typescript
// Fetch total count and max complexity
const { count, error: countError } = await supabase
  .from('saved_queries')
  .select('*', { count: 'exact', head: true });

const { data: maxComplexity, error: maxError } = await supabase
  .from('saved_queries')
  .select('complexity')
  .order('complexity', { ascending: false })
  .limit(1)
  .single();
```

### 6.5 DELETE — Remove a Saved Query

```typescript
// Delete a specific saved query by ID
const { error } = await supabase
  .from('saved_queries')
  .delete()
  .eq('id', queryId);
// RLS ensures users can only delete their own rows
```

## 7. Data Validation

### 7.1 Application-Level Validation (Zod)

```typescript
import { z } from 'zod';

export const savedQuerySchema = z.object({
  query_text: z
    .string()
    .min(1, 'Query text is required')
    .max(5000, 'Query exceeds maximum length'),
  explanation: z
    .string()
    .min(1, 'Explanation is required'),
  complexity: z
    .number()
    .int()
    .min(1, 'Complexity must be at least 1')
    .max(10, 'Complexity must be at most 10'),
  optimization_tips: z
    .string()
    .nullable()
    .optional(),
});

export type SavedQueryInput = z.infer<typeof savedQuerySchema>;
```

### 7.2 Database-Level Constraints

| Constraint | Column | Type | Rule |
|---|---|---|---|
| PRIMARY KEY | `id` | `UUID` | Unique, auto-generated |
| NOT NULL | `user_id`, `query_text`, `explanation`, `complexity`, `created_at` | Various | Must not be null |
| FOREIGN KEY | `user_id` | `UUID` | Must reference `auth.users(id)` |
| CASCADE DELETE | `user_id` | FK | If user is deleted, all their queries are deleted |
| CHECK | `complexity` | `INTEGER` | Must be between 1 and 10 |

| Data Type | Retention | Deletion Trigger |
|---|---|---|

| Orphaned data | Automatic cleanup | CASCADE on user deletion |

Monitor storage usage via Supabase Dashboard → Database → Usage. Set up alerts if approaching 400MB (80% of free tier).

## 10. Backup & Recovery

### 10.1 Supabase Backup Schedule

| Feature | Free Tier | Pro Tier |
|---|---|---|
| Automatic backups | Daily | Daily |
| Backup retention | 7 days | 7 days (configurable) |
| Point-in-time recovery | No | Yes |
| Manual backup/export | Via pg_dump | Via pg_dump or dashboard |

### 10.2 Manual Backup Procedure

```bash
# Export database using pg_dump
pg_dump -h <supabase-host> -U postgres -d postgres \
  --schema=public \
  --no-owner \
  --no-privileges \
  -f sqlsense_backup_$(date +%Y%m%d).sql
```
